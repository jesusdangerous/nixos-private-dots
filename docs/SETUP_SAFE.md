# Безопасная настройка и обновление

Цель: минимизировать риск сломанной системы и ускорить восстановление.

## 1) Перед первым применением

- Проверь, что в `flake.nix` заполнен блок `user`.
- Проверь, что диск-модули (`modules/nixos/filesystems.nix`, `modules/nixos/hibernate.nix`) либо корректно настроены, либо не подключены в `modules/nixos/bundle.nix`.
- Убедись, что секреты и внешние пути не захардкожены под чужой `home`.

## 2) Безопасный цикл применения

1. Сборка без переключения:
```sh
sudo nixos-rebuild build --flake .#nixos
```
2. Тестовый запуск в текущей сессии:
```sh
sudo nixos-rebuild test --flake .#nixos
```
3. Если всё стабильно, постоянное применение:
```sh
sudo nixos-rebuild switch --flake .#nixos
```

Если `test` вызывает нестабильность, используй:
```sh
sudo nixos-rebuild boot --flake .#nixos
```
и перезагрузку.

## 3) Базовый smoke-test после ребилда

- Логин в niri с SDDM.
- Открываются `alacritty`, `rofi`, `thunar`.
- Работают горячие клавиши звука/яркости.
- Открывается браузер по `Mod+B`.
- При необходимости стартует VPN клиент `flclash`.

## 4) Быстрый откат

- В меню загрузки выбрать предыдущее поколение NixOS.
- После входа зафиксировать откат:
```sh
sudo nixos-rebuild switch --rollback
```

## 5) Гигиена конфигурации

- Не редактируй сразу много подсистем за один коммит.
- Держи один источник правды для пользователя и путей (через `user` в `flake.nix`).
- Старайся держать документацию в `docs/`, а статические ресурсы в `assets/`.
